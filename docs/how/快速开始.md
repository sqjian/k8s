## 1、nginx

### NodePort ###

**nginx-rc.yaml**

```yaml
# 此部分描述的是rc文件本身的信息
apiVersion: v1
kind: ReplicationController
metadata:
  name: nginx
spec:
  replicas: 1
  selector:
    # 此部分需与下面labels有重合，否则rc无法创建成功，报错
    app: nginx
  template:
    # 此部分描述的是pod的信息
    # 虽然在命名上和rc会有重复，但由于所描述的非同一维度，故不影响
    metadata:
      labels:
        app: nginx
    spec:
      hostNetwork: true
      containers:
        - name: nginx
          image: nginx
          ports:
            - containerPort: 80

```

**nginx-svc.yaml**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx
spec:
  # 以NodePort方式暴露端口，NodePort 默认范围是 30000-32767，可通过向apiserver添加参数 –service-node-port-range=1-65535
  type: NodePort
  ports:
    - port: 80
      nodePort: 80
  selector:
    app: nginx

```

## 2、mysql

### hostNetwork ###

**nginx-deployment.yaml**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mysql
  labels:
    app: mysql
spec:
  replicas: 1
  selector:
    matchLabels:
      app: mysql
  strategy:
    type: Recreate
  template:
    metadata:
      labels:
        app: mysql
    spec:
      hostNetwork: true
      containers:
        - env:
            - name: MYSQL_ROOT_PASSWORD
              value: "12345678"
          image: mysql:5.7
          name: mysql
          volumeMounts:
            - name: mysql-path
              mountPath: /var/lib/mysql
      volumes:
        - name: mysql-path
          hostPath:
            path: /data1/container/mysql/data
```
